# AI 工作室架构说明

## 📐 模块定位

AI 工作室是**独立一级模块**，与笔记、创作、成长同级。

```
导航结构：
├── 仪表盘
├── 笔记（笔记列表管理）
├── 创作（新建笔记）
├── 🤖 AI 工作室 ← 独立模块
├── 成长（数据分析）
└── 订阅
```

---

## 🚪 两个入口

### 入口 1：从 AI 工作室进入（主入口）

**路径：** `/dashboard/ai-studio`

**流程：**
```
AI 工作室主页 → 选择笔记 → 选择平台 → 生成多版本 → 对比/发布
```

**页面功能：**
- 显示所有笔记列表
- 支持搜索/筛选
- 按平台筛选（已生成版本的笔记）
- 点击笔记进入改写详情页

**文件：**
- `/app/dashboard/ai-studio/page.tsx`

---

### 入口 2：从单篇笔记进入（快捷入口）

**路径 A：笔记列表页**
```
笔记列表 → 点击"🤖 AI 改写" → AI 工作室主页
```

**路径 B：笔记详情页**
```
笔记详情 → 点击"🤖 AI 改写" → AI 工作室改写页（带笔记 ID）
```

**新增按钮位置：**
1. 笔记列表页顶部：新增"AI 改写"按钮
2. 笔记详情页顶部：新增"🤖 AI 改写"按钮（在编辑按钮左侧）

**文件：**
- `/app/dashboard/notes/page.tsx`（列表页）
- `/app/dashboard/notes/[id]/page.tsx`（详情页）

## 🗄️ 数据流

### 从笔记到 AI 版本

```
笔记 (notes 表)
   ↓
用户选择平台 + 风格
   ↓
调用 AI 生成 API
   ↓
保存到 ai_versions 表
   ↓
返回到前端展示
```

### 数据表关系

```
notes (笔记表)
  └── ai_versions (AI 版本表)
       ├── wechat_publications (公众号发布)
       ├── xiaohongshu_publications (小红书发布)
       ├── short_video_projects (短视频项目)
       ├── podcast_episodes (播客节目)
       └── feedback_loops (数据反馈)
```

## 🎯 使用流程示例

### 场景 1：从 AI 工作室开始
1. 点击左侧导航 "🤖 AI 工作室"
2. 浏览笔记列表，找到想改写的笔记
3. 点击笔记卡片
4. 选择平台（如"公众号"）
5. 生成 1-3 个版本
6. 对比选择最佳版本
7. 复制/发布

### 场景 2：从笔记详情开始
1. 在笔记列表打开某篇笔记
2. 点击顶部 "🤖 AI 改写" 按钮
3. 直接进入改写页（已选中该笔记）
4. 选择平台并生成版本
5. 对比选择最佳版本

## 📊 数据隔离

AI 工作室是**独立模块**，有以下特点：

1. **独立页面** - 不依赖笔记页面
2. **独立数据** - AI 版本存储在独立表
3. **独立流程** - 有自己的生成/对比/发布流程
4. **可回溯** - 可以从笔记跳转到 AI 工作室，也可以从 AI 工作室选择笔记
